# 01_Creative_Direction.md

# Creative Direction Bible

Version 1.0

## Purpose

This document defines the artistic identity of the portfolio. It governs
visual storytelling, cinematography, composition, pacing, and emotional
progression. Whenever there is uncertainty about how something should
look or feel, this document is authoritative.

------------------------------------------------------------------------

# Creative Objective

The visitor should feel like they are exploring a carefully directed
interactive short film instead of browsing a website.

The experience should communicate:

-   Precision
-   Ambition
-   Curiosity
-   Craftsmanship
-   Discipline

The portfolio must never feel like a template, a dashboard, or a
slideshow.

------------------------------------------------------------------------

# Design Influences

Draw inspiration from:

-   Awwwards Site of the Day winners
-   High-end title sequences
-   Editorial magazine layouts
-   Apple product films
-   Interactive museum installations
-   Architectural visualization

Avoid copying any one website. Extract principles rather than
aesthetics.

------------------------------------------------------------------------

# Narrative

The story is:

"I think deeply. I build carefully. I validate with achievement. I
persist through discipline."

Every chapter should reinforce that story.

------------------------------------------------------------------------

# Cinematic Language

## Camera

The camera is an invisible narrator.

Rules:

-   Never jerk.
-   Never snap.
-   Ease naturally.
-   Use long cinematic movements.
-   Prefer camera movement over object movement.

## Composition

Each viewport should contain:

-   foreground
-   midground
-   background

Every important object should have breathing room.

Negative space is a deliberate design element.

------------------------------------------------------------------------

# Motion Philosophy

Motion should feel physical.

Every animated object must appear to possess:

-   mass
-   momentum
-   inertia
-   drag
-   friction
-   damping

Avoid linear movement whenever possible.

Preferred easing:

-   Custom cubic curves
-   Spring systems
-   Elastic settling only when appropriate

------------------------------------------------------------------------

# Lighting Philosophy

White scenes

-   diffuse light
-   soft shadows
-   clean reflections

Dark scenes

-   directional key light
-   subtle rim lighting
-   restrained bloom
-   physically believable reflections

Lighting should direct attention instead of relying on bright colors.

------------------------------------------------------------------------

# Color Story

White = clarity

Black = exploration

Blue = continuity

Gold = achievement

Red = determination

Accent colors must remain rare so they retain impact.

------------------------------------------------------------------------

# Typography

Headlines are oversized and confident.

Body copy is concise.

No paragraph should explain what motion can communicate visually.

Handwritten annotations should appear like notes from a design notebook
rather than decoration.

------------------------------------------------------------------------

# Interaction Philosophy

Scrolling is the timeline.

Hover interactions should reveal depth, not merely scale objects.

Clicks should be meaningful and reserved for exploration rather than
navigation.

------------------------------------------------------------------------

# Sound (Optional)

If audio is introduced:

-   subtle
-   spatial
-   optional
-   easily muted

Never autoplay loud music.

------------------------------------------------------------------------

# Accessibility

Motion-sensitive users should be able to reduce animation intensity.

Text contrast must always meet accessibility requirements.

The narrative must remain understandable even with reduced motion
enabled.

------------------------------------------------------------------------

# Visual Continuity

A persistent cerulean line travels throughout the experience.

Chapter mapping:

Arrival -\> ink stroke

Recognition -\> medal ribbon

Creation -\> circuitry

Vault -\> archival binding

Endurance -\> marathon route

This element visually ties every chapter together.

------------------------------------------------------------------------

# Acceptance Criteria

The experience should:

-   feel handcrafted
-   feel cinematic
-   avoid generic UI
-   reward slow exploration
-   remain readable
-   prioritize quality over quantity

If a new feature weakens these goals, it should be rejected regardless
of technical sophistication.
