# 00_Project_Vision.md

# Systems & Endurance

## Master Vision Document

**Version:** 1.0

------------------------------------------------------------------------

# Purpose

This document defines the philosophy, goals, constraints, and success
criteria for the entire portfolio. It is the highest-priority
specification in the project. Every future document must comply with the
principles defined here.

The website is not intended to function as a traditional portfolio. It
is an interactive, cinematic experience that communicates personality,
discipline, engineering ability, creativity, and ambition through
motion, spatial design, and storytelling.

The objective is for visitors to remember the experience first and the
projects second.

------------------------------------------------------------------------

# Primary Goal

Create a portfolio that feels comparable to top Awwwards winners while
remaining readable, performant, and purposeful.

Visitors should experience a continuous narrative rather than a sequence
of unrelated pages.

The website should answer one question:

> "Who is this person?"

without requiring lengthy paragraphs.

------------------------------------------------------------------------

# Core Identity

The portfolio expresses four pillars:

1.  Systems Thinking
2.  Engineering Excellence
3.  Design Craftsmanship
4.  Endurance & Discipline

Every section must reinforce at least one of these pillars.

------------------------------------------------------------------------

# Emotional Journey

## Arrival

Curiosity. Minimalism. Silence. Precision.

## Recognition

Confidence. Achievement. Momentum.

## Creation

Energy. Innovation. Complexity. Color.

## Evidence

Trust. Credibility. Professionalism.

## Endurance

Determination. Resilience. Calm confidence.

------------------------------------------------------------------------

# Story Structure

The website is divided into chapters rather than sections.

1.  Arrival
2.  Recognition
3.  Creation
4.  Vault
5.  Endurance

Objects transition naturally between chapters. Abrupt cuts should be
avoided.

------------------------------------------------------------------------

# Design Philosophy

## Less but Better

Every element must have a reason to exist.

If removing an element improves clarity, remove it.

## Motion Communicates Meaning

Animation is never decorative.

Motion must indicate:

-   weight
-   direction
-   hierarchy
-   emphasis
-   continuity

## Spatial Interfaces

Content should exist inside a three-dimensional space whenever doing so
improves storytelling.

3D should never exist solely because it looks impressive.

------------------------------------------------------------------------

# Visual Language

The experience alternates between bright white and deep black
environments.

White symbolizes clarity and thought.

Black symbolizes depth and exploration.

Cerulean Blue (#3A86FF) acts as the continuous visual thread that
connects every chapter.

------------------------------------------------------------------------

# Audience

Primary audiences include:

-   university admissions officers
-   scholarship reviewers
-   professors
-   recruiters
-   engineers
-   designers
-   collaborators

The portfolio should communicate equally well to both technical and
non-technical visitors.

------------------------------------------------------------------------

# Success Criteria

A successful implementation should cause visitors to:

-   understand the narrative without explanation
-   remember at least three projects
-   remember the running theme
-   perceive exceptional attention to detail
-   perceive engineering maturity

------------------------------------------------------------------------

# Non-Negotiable Rules

-   No template appearance.
-   No stock interactions.
-   No generic cards replacing specified experiences.
-   No unnecessary scrolling effects.
-   No autoplay video backgrounds.
-   No excessive neon or cyberpunk aesthetics.
-   Maintain 60 FPS whenever possible.
-   Accessibility must not be sacrificed for visual spectacle.

------------------------------------------------------------------------

# Technical Direction

Preferred stack:

-   HTML
-   CSS
-   JavaScript
-   GSAP
-   ScrollTrigger
-   Lenis
-   Three.js
-   GLSL where appropriate

Avoid heavy frameworks unless they provide measurable value.
